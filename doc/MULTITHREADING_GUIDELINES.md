# MonsterWar 多线程实践指南

> 目标：把项目里已经落地的并行化方案整理成可执行的约束，后续按优先级逐项实现。

## 1. 优先级

1. 先做“只读任务”的后台化：文件读取、JSON 解析、资源路径扫描。
2. 再做“主线程提交”的优化：资源注册、场景创建、ECS 写入、UI 更新。
3. 最后处理生命周期：任务取消、关闭回收、错误回传、日志统一。

## 2. 项目内的硬规则

- 后台线程只处理纯数据，不直接碰 `entt::registry`、`Renderer`、`ImGui`、`SceneManager`。
- 所有 SDL 资源创建、场景切换、UI 创建销毁都保留在主线程。
- 异步任务返回明确状态，不只依赖裸 `bool`。
- 任务完成后由主线程统一提交结果，避免共享可变状态。
- 场景关闭或应用退出时必须等待或回收未完成任务。

## 3. 推荐实现顺序

### 3.1 资源映射

- 后台解析 `assets/data/resource_mapping.json`。
- 主线程在初始化末尾执行预加载。
- 异常与失败原因要显式传递出来。

### 3.2 关卡加载

- 后台读取 `tmj/tsj` 与关卡配置。
- 主线程把解析结果写入场景和 ECS。

### 3.3 存档写入

- 存档序列化与落盘放后台。
- 退出前等待任务结束。

## 4. 本仓库当前最值得继续做的点

- 资源映射异步任务的错误状态结构化。
- 异步任务关闭流程统一化。
- 关卡与存档任务的结果对象标准化。

## 5. 验证标准

- 编译通过。
- 场景切换不回归。
- 退出时不遗留后台任务。
- 失败时日志能直接定位到具体阶段和资源路径。
