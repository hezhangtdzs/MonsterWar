# MonsterWar 项目计划表

> **目标**: 在现有引擎与 ECS 基础上，逐步完善为完整可玩的塔防游戏。  
> **执行方式**: 每次只推进一个小阶段；完成后同步更新 `doc/PROJECT_PROGRESS.md`。

---

## 1. 总体计划

| 阶段 | 目标 | 主要产出 | 状态 |
|------|------|----------|------|
| 0 | 现状统计 | 进度统计与计划文档 | 已完成 |
| 1 | 基础对齐 | 统一启动流程、场景初始化、配置入口 | 已完成 |
| 2 | 塔防核心闭环 | 建造、升级、波次、经济、基地生命、胜负条件 | 进行中 |

| 4 | UI 与交互 | 主菜单、暂停、波次提示、结算界面、操作引导 | 进行中 |

## 2. 阶段拆解

### 阶段 1：基础对齐

- 整理 `GameApp` 启动流程与初始场景切换。
- 统一 `GameScene` 的初始化、更新、清理职责。
- 明确资源加载、蓝图加载、地图加载的顺序。
- 将当前实现状态同步到文档。

### 阶段 2：塔防核心闭环

- 增加敌人波次控制与刷怪节奏。
- 增加基地生命、敌人到达基地后的扣血逻辑。
- 增加建造、放置、升级、出售等塔防核心操作。
- 增加资源/金币等基础经济系统。
- 当前已完成基础波次刷怪、基地扣血、金币扣费、基础升级入口和基础出售入口，后续继续补齐胜负 UI 与更精细的建造面板。
- 已接入出击准备与放置确认闭环：底部肖像可进入准备态，左键确认后创建地图单位并占用放置点，右键可取消准备，远程单位支持范围预览。
- 已统一主要运行时日志为位置化输出，便于后续调试波次和战斗流程。
- 已接入共享关卡状态 `GameStats` 与 `GameRuleSystem`，`cost` 资源改由规则系统推进并供 UI 遮罩读取。

### 阶段 3：战斗系统完善

- 打通目标锁定、攻击启动、伤害结算、死亡清理。
- 完善阻挡、治疗、范围攻击、特殊技能等规则。
- 统一动画结束后的状态回切逻辑。
- 加强事件定义与事件流转的可读性。
- 已确认战斗事件链路闭合，且主要战斗系统日志已统一为位置化输出。
- 已补齐远程攻击帧 `emit` 的结算分流，弓箭手等远程单位可正常命中目标。

### 阶段 4：UI 与交互

- 增加主菜单、暂停、游戏胜负、波次提示界面。
- 增加鼠标悬停、选中、建造预览等交互反馈。
- 增加基础操作说明与错误提示。
- 已接入纹理 HUD、波次提示、暂停遮罩与 ImGui 调试层，暂停与恢复可通过按钮或 `pause` 动作切换。
- 已使用 `assets/textures/UI` 下的资源构建更完整的游戏内 HUD 与暂停面板。
- 已接入出击准备界面反馈：准备态单位可按合法性切换颜色，远程准备态会显示攻击范围圆。
- 已将肖像条的锁定反馈改为读取共享 `GameStats.cost_`，保证资源与 UI 状态一致。

### 阶段 5：内容补齐

- 扩充敌人、塔、防御技能、地图与路径配置。
- 为不同关卡设计不同的难度曲线。
- 完善蓝图配置与数据驱动内容。
- 已整理 `assets` 资源驱动需求清单，便于后续按资源优先级补齐内容与渲染。
- 已将 `buildings.tsj` 的 `place` 属性解析为近战/远程建造位标签，场景层已能区分可建造位置语义。
- 已将 `GameScene` 的关卡地图、准备时间和波次内容切换为 `level_config.json` 数据驱动。
- 已接入 `default_session_data.json` 和 `ui_config.json`，底部角色肖像条按跨关卡数据与 UI 配置生成，并支持排序与点击。
- 已接入单位死亡特效与世界空间血条，死亡时会触发特效，单位生命值会以血条形式显示。

### 阶段 6：表现与优化

- 补齐音效、音乐与动画特效。
- 在主循环中补充进度统计与必要日志。
- 对派发、更新和渲染关键路径做性能检查。
- 已接入远程投射物与治疗特效的临时视觉实体，并采用对象池式复用降低频繁创建开销。

### 阶段 7：测试与发布准备

- 为关键系统补充回归测试。
- 检查资源引用、文档链接与启动步骤。
- 整理发布前检查清单。

---

## 3. 执行规则

1. 每一步只做一个明确目标。
2. 每完成一步就更新 `doc/PROJECT_PROGRESS.md`。
3. 如果实现范围发生变化，先修订计划表，再继续执行。
4. 新增功能优先保持数据驱动，尽量复用现有 ECS 结构。
